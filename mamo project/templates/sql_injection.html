üß† How SQL Injection Happens

A login form takes input from the user:

username = admin' OR '1'='1
password = anything

Then the vulnerable server builds the SQL like this: 


SELECT * FROM users

WHERE username = 'admin' OR '1'='1'

AND password = 'anything'

‚ùó Because of the OR '1'='1', the database thinks the login is correct for ANY password.
The server checks: "Is username admin OR 1=1?" ‚ûî 1=1 is always true ‚ûî login bypassed!
